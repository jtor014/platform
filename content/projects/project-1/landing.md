# Project 1: The Static Site

"I can direct AI to build a website and deploy it to a real URL."

## What you'll build

A professional business website for GreenScape Landscaping — a small landscaping company in Melbourne. Hero section, services grid, testimonials, about section, contact details. Single page, responsive, deployed to a live URL.

## What's new (the +1)

Everything. This is your first project. The terminal, VS Code, Git, Claude Code — all new. The product is deliberately simple so all your attention goes to learning the loop.

## Why this project is deliberately simple

GreenScape is a single-page static website. No database, no backend, no user accounts. That's intentional — there's enough to learn just from the tools and the workflow. If the product were complex, you'd be distracted by business logic when you should be learning how to direct AI, read a pull request, and verify output against a spec.

## What success looks like

When you're done, you should have:
- A live URL you can share with anyone
- A site that matches the design specification (check with DevTools, not by eye)
- No features that aren't in the PRD (you caught every scope creep attempt)
- Confidence in the loop: branch → direct → verify → merge. You could do it again without the guide

## What you'll learn

- Direct Claude Code through the complete ticket cycle: branch → implement → test → review → merge
- Verify AI output against a design specification using browser DevTools
- Catch anti-scope violations (AI adding features you didn't ask for)
- Deploy to a live URL using Netlify and handle deploy failures
- Read and act on AI code reviews (Claude self-review + Gemini independent review)

## Prerequisites

- Completed shared setup (VS Code, Git, GitHub CLI, Claude Code)
- Completed Web Dev setup (Node.js, Netlify account)

## Estimated time

2-4 days. 5-7 tickets.
